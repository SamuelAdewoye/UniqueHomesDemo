# Output & Token Efficiency Guidelines
- **Be Extremely Concise:** Provide short, direct answers. Do not summarize what you did unless explicitly asked.
- **No Unnecessary Chatter:** Skip introductory or concluding polite filler (e.g., "Sure, I can help with that!").
- **Code Only for Edits:** When writing or refactoring code, output ONLY the modified code or clean diffs without repeating unchanged file contents.
- **No Unused Imports:** Always keep code outputs clean to prevent re-prompting loops.
